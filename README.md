## 2. 设计目的
本次课程设计旨在通过实现哈夫曼树密码加密与解密系统，深入理解哈夫曼编码的原理及其在数据压缩和加密领域的应用。具体目的包括：
1. 掌握哈夫曼树的构建原理和算法实现；
2. 理解变长编码的思想及其在数据压缩中的优势；
3. 掌握利用哈夫曼树进行数据加密和解密的方法；
4. 熟悉文件操作在数据存储中的应用；
5. 提升综合运用 C++ 语言解决实际问题的能力。

**哈夫曼编码**是一种变长前缀编码，其核心思想是为出现频率高的字符分配较短的编码，为出现频率低的字符分配较长的编码，从而达到数据压缩的目的。这种编码方式在数据压缩、信息传输和密码学等领域有着广泛的应用。

## 3. 设计课题与功能

**课题名称：** 哈夫曼树密码加密与解密系统

本系统主要实现以下功能：
1. 统计输入字符串中各字符的出现频率；
2. 基于字符频率构建哈夫曼树；
3. 生成哈夫曼编码表；
4. 使用哈夫曼编码对字符串进行加密；
5. 使用哈夫曼树对编码后的字符串进行解密；
6. 将加密和解密后的结果保存到文件中；
7. 提供友好的交互界面，方便用户操作。

## 4. 总体设计方案

系统架构：
本系统采用模块化设计，主要包含以下几个模块：
1. **哈夫曼树构建模块**：负责根据字符频率构建哈夫曼树；
2. **编码生成模块**：负责从哈夫曼树生成各字符的哈夫曼编码；
3. **加密模块**：负责使用哈夫曼编码对输入字符串进行加密；
4. **解密模块**：负责使用哈夫曼树对编码后的字符串进行解密；
5. **文件操作模块**：负责将加密和解密结果保存到文件；
6. **用户交互模块**：负责提供菜单界面，处理用户输入。

## 5. 详细设计

### 5.1 哈夫曼树构建模块
- **功能**：根据输入文本中字符的频率分布，构建最优二叉树（哈夫曼树）。
- **数据结构**：
  1. 使用 `HuffmanNode` 结构体表示树的节点，包含字符数据、频率及左右子节点指针。
  2. 采用最小优先队列（小顶堆）存储节点，确保每次能快速取出频率最小的两个节点。

### 5.2 编码生成模块
- **功能**：遍历哈夫曼树，为每个字符生成对应的二进制编码。
- **算法思路**：
  1. 从根节点开始递归遍历；
  2. 向左子树遍历路径标记为 `0`；
  3. 向右子树遍历路径标记为 `1`；
  4. 当到达叶节点时，记录当前路径作为该字符的编码；
  5. 使用哈希表（`unordered_map`）存储字符与编码的映射关系。

### 5.3 加密模块
- **功能**：根据编码表将原始字符串转换为二进制序列。
- **实现步骤**：
  1. 遍历输入字符串的每个字符；
  2. 从编码表中查找对应字符的二进制编码；
  3. 按顺序拼接所有字符的编码，生成最终的加密字符串。

### 5.4 解密模块
- **功能**：利用哈夫曼树将二进制序列还原为原始字符串。
- **算法思路**：
  1. 从哈夫曼树的根节点开始，按二进制序列的每一位进行遍历；
  2. 遇到 `0` 则向左子树移动；
  3. 遇到 `1` 则向右子树移动；
  4. 当到达叶节点时，记录该节点的字符，并重置到根节点继续遍历。

### 5.5 文件操作模块
- **功能**：实现加密 / 解密结果的文件读写。
  1. **保存加密结果**：将二进制编码写入 `encrypted.txt` 文件。
  2. **保存解密结果**：将还原的原始文本写入 `decrypted.txt` 文件。

### 5.6 用户交互模块
- **功能**：提供菜单界面，处理用户输入，协调各模块工作。

## 6. 课程设计总结

在这次哈夫曼树加密解密系统的课程设计中，我收获颇丰。起初，我对如何将哈夫曼树理论转化为实际代码感到迷茫，但通过一步步拆解问题，逐渐理清了思路。我先构建哈夫曼树模块，利用优先队列实现节点合并，却在节点排序上遇挫，调试后才明白比较函数的关键作用。编码生成时，递归遍历的逻辑让我反复推敲，最终通过前缀字符串传递路径信息解决了编码错乱问题。

测试阶段，我发现文件操作和输入处理存在漏洞，比如文件打开失败未提示、输入缓冲区残留换行符等，这些细节让我意识到工程化思维的重要性。当看到加密后的二进制序列能准确还原为原始字符串时，成就感油然而生。这次设计不仅加深了我对数据结构的理解，更让我体会到模块化设计和错误处理的必要性。未来，我想进一步优化压缩效率，尝试将哈夫曼编码与其他加密技术结合，拓展更多应用可能。
